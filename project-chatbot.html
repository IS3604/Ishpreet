◆ Overview

Where QA Meets Frontend Development

The WingmanX Chatbot was born at the intersection of two worlds I work in daily: QA testing and frontend development. As part of the WingmanX mobile app team at Rewise Analytics, I understood the product deeply — its flows, its edge cases, its user journeys. So I built a chatbot interface that simulates those conversational flows in the browser.

The chatbot runs entirely on a mock API, meaning there's no backend dependency — it works anywhere, anytime. The conversation logic was designed with prompt engineering techniques, mapping realistic user inputs to structured response flows.

Built from QA insight: Months of testing WingmanX gave me a deep map of every user flow and failure mode. I used that knowledge to design conversation paths that feel real — because they're based on real usage patterns I observed as a QA engineer.

◆ Features

What Makes It Work

  • Real-time conversational UI with smooth message rendering and response timing that feels natural
  • Mock API integration — all conversation logic lives client-side, making it fast and dependency-free
  • Prompt-engineered response flows designed from actual WingmanX user journey knowledge
  • Covers multiple conversation branches — greetings, feature queries, error states, and fallback responses
  • Responsive design that works across mobile and desktop — reflecting the mobile-first product it mirrors
  • Deployed on GitHub Pages for zero-friction access
◆ The Connection to QA

Why a QA Engineer Built This

As a QA engineer on WingmanX, I wrote test cases for every flow, every edge case, every failure scenario. That work gave me an unusually detailed mental model of how users interact with the product. Building this chatbot was a way to express that model in code.

It also expanded my skill set — from testing interfaces to building them. Understanding both sides makes me a better QA engineer: I know what it costs to build a feature, which makes me more thoughtful about what I flag as a defect versus what's acceptable.
